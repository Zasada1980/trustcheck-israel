**Дата:** 22 декабря 2025  
**Тип документа:** Technical & Business Audit  
**Статус:** Strategic Planning Document

## Executive Summary

Данный документ представляет независимый технический и бизнес-аудит проекта **Search Business Firms (SBF)** — платформы для поиска информации о бизнесах в Израиле. На основе анализа государственных источников данных, коммерческих платформ и законодательной базы предложена оптимальная архитектура, карта взаимодействия пользователей и оценка рисков.

**Ключевые выводы:**
- ✅ Прямой доступ к государственным данным **технически возможен** (с ограничениями)
- ⚠️ Юридические барьеры требуют регистрации Database License (₪15K-50K)
- ✅ Гибридная модель (государственные API + коммерческие партнёры) оптимальна
- 💰 Break-even достижим при 500 отчётах/месяц

## Часть I: Анализ источников данных

### 1.1. Государственные источники (Layer 0)

#### רשם החברות (Companies Registrar) — Регистратор компаний

**Доступность:** ⭐⭐⭐⭐⭐ (Высокая)

**Технический доступ:**
- **API:** REST/JSON через data.gov.il (CKAN архитектура)
- **Форматы:** JSON (real-time), CSV/XML (bulk dumps)
- **Аутентификация:** Не требуется для базовых данных
- **Rate limits:** Нет официальных ограничений для базовых запросов

**Данные:**
```json
{
  "company_id": "515123456",
  "company_hebrew_name": "חברת בדיקה בע״מ",
  "company_english_name": "Test Company Ltd",
  "company_status": "Active|Dissolved|Violating",
  "company_exists": true,
  "corporation_type": "Private|Public|Government"
}
```

**Стоимость доступа (2025):**
- Базовая валидация: **Бесплатно**
- Полная выписка (Nesach Maleh): **₪11-36** за документ
- Bulk API (коммерческий): Индивидуальный договор

**Ограничения:**
- ❌ Финансовые отчёты частных компаний не публичны
- ❌ Глубокая структура акционеров только в платных выписках
- ⚠️ Массовая агрегация данных о физлицах требует Database License

**Применимость для SBF:** ⭐⭐⭐⭐⭐ (Критичный источник)

---

#### רשות המסים (Tax Authority) — Налоговое управление

**Доступность:** ⭐⭐⭐⭐ (Высокая после регистрации)

**Технический доступ:**
- **API:** OAuth2, REST/JSON через openapi-portal.taxes.gov.il
- **Регистрация:** "Программный дом" (Beit Tochna) — обязательно
- **Аутентификация:** Access Token (JWT), Refresh Token
- **Инфраструктура:** Требуется Smart Card для админа + статические IP

**Процесс регистрации Программного дома:**
1. Административная регистрация в Shaam (система налоговой)
2. Получение Smart Card (Comsign/PersonalID) — ₪400/год
3. Создание Organization на портале openapi-portal.taxes.gov.il
4. Подписание декларации информационной безопасности
5. Вайтлистинг IP-адресов серверов

**Ключевой API для SBF: Проверка статуса Osek Murshe**

Endpoint: `/api/v1/ishur-nihul-sfarim`
```json
Request: {
  "dealer_number": "012345678"
}

Response: {
  "status": "Active|Closed|Suspended",
  "business_name": "עסק לדוגמא",
  "tax_withholding_rate": 0,  // 0% = идеально, >0% = риск
  "valid_until": "2025-12-31"
}
```

**Стоимость:**
- Setup: Время (бюрократия) + Smart Card ₪400
- Per Query: **Бесплатно** (после регистрации)

**Преимущества:**
- ✅ Real-time данные
- ✅ Легальный доступ без посредников
- ✅ Нет переменных расходов per-query

**Применимость для SBF:** ⭐⭐⭐⭐⭐ (Критичный источник для עוסקים)

---

#### נט המשפט (Net HaMishpat) — Судебная система

**Доступность:** ⭐ (Очень низкая)

**Статус:** Закрытая система, **НЕТ публичного API**

**Барьеры доступа:**
1. Smart Card (₪400/год) — только для адвокатов
2. Национальная идентификация — только свои дела
3. Скрейпинг запрещён ToS (юридический риск)
4. Прецедент "Hashavim HPS" — суд не обязал открыть API

**Уровни доступа:**
- Внутренний: Судьи, секретариат
- Адвокаты: Smart Card + VPN
- Граждане: Только свои дела через Hizdahut Leumit
- Публика: Выборочная публикация решений (нет поиска)

**Альтернативы для SBF:**

**Вариант A: User-Generated Access**
```
Пользователь → Логин в Net HaMishpat (Hizdahut Leumit) 
            → SBF проксирует сессию 
            → Извлекает справку об отсутствии дел
```
- ✅ Легально (доступ уполномоченным лицом)
- ⚠️ Требует доверия пользователя
- ⚠️ Сложная UX

**Вариант B: Партнёрство с Takdin/PsakDin**
- Использовать API частных агрегаторов
- Стоимость: ₪99-299/месяц подписка
- ✅ Легально, ✅ Надёжно

**Рекомендация для SBF:** Вариант B (партнёрство), Вариант A для премиум-сегмента.

**Применимость для SBF:** ⭐⭐ (Только через посредников/пользователей)

---

#### בנק ישראל (Bank of Israel) — Ограниченные счета

**Доступность:** ⭐⭐⭐ (Средняя)

**Технический доступ:**
- **НЕТ API** — только файлы
- **Формат:** ASCII structure / Excel (ZIP)
- **Обновление:** Ежедневно
- **Разделение:** Корпорации / Частные лица

**Технология доступа: ETL Pipeline**
```
Daily Cron Job → Download ZIP from boi.org.il
              → Unzip & Parse (ISO-8859-8/Windows-1255)
              → Update Redis cache
              → Validate freshness (lawsuit risk)
```

**Юридический риск:**
- 🔴 Публикация устаревших данных = иск о диффамации
- Обновление должно быть синхронным с БИ

**Стоимость:**
- Доступ: **Бесплатно**
- Инфраструктура: ETL скрипт (dev time)

**Применимость для SBF:** ⭐⭐⭐⭐ (Важный индикатор риска)

## Часть II: Юридические и регуляторные барьеры

### 2.1. Database License (רישיון מאגר מידע)

**Требование:** Обязательно для SBF Platform

**Триггеры лицензирования:**
1. ✅ База содержит >10,000 физлиц
2. ✅ База содержит "чувствительные данные" (экономическое положение, долги)
3. ✅ Используется для коммерческих целей

**Процесс получения:**
1. Подача заявки в Управление защиты конфиденциальности (PPA)
2. Описание целей использования базы
3. Декларация мер безопасности
4. Согласие на аудит

**Стоимость:**
- Регистрация: **₪15,000-50,000** (одноразово)
- Ежегодная пошлина: **₪5,000**

**Санкции за несоблюдение:**
- Уголовная ответственность (до 1 года тюрьмы)
- Штрафы до ₪500,000

**Срок получения:** 2-6 месяцев

---

### 2.2. Privacy Protection Law (חוק הגנת הפרטיות)

**Принципы:**
1. **Purpose Limitation** — данные только для заявленной цели
2. **Data Minimization** — собирать минимум необходимого
3. **User Consent** — явное согласие на агрегацию

**Критические риски для SBF:**
- ❌ Массовый скрейпинг судебных дел для "чёрных списков"
- ❌ Профилирование физлиц без согласия
- ❌ Хранение данных дольше необходимого

**Решение:**
```
User Registration → Explicit Consent Form
                 → "Я разрешаю SBF собирать данные обо мне 
                    из государственных источников для проверки"
```

---

### 2.3. Open Banking License (רישיון ספק מידע פיננסי)

**Если SBF планирует анализ банковских транзакций:**

**Требования:**
- Минимальный капитал: **₪200,000**
- Соответствие Berlin Group NextGenPSD2
- ISO 27001 сертификация

**Преимущества vs Кредитное бюро:**
- ✅ Меньший капитал (vs ₪500K-4M для кредитного бюро)
- ✅ Прямой доступ к банковским данным (Leumi, Hapoalim)
- ✅ С согласия пользователя

**Применимость:** Tier 3 (премиум feature для проверки ИП)

### 3.2. Модули платформы (детально)

#### Module 1: Business Validator (Валидатор бизнесов)

**Источники:**
- Companies Registrar API (gov.il)
- Shaam Tax Authority API

**Функции:**
1. **Quick Check (Быстрая проверка)**
   - Input: H.P. number или название
   - Output: Существует ли, статус, базовые данные
   - Latency: <500ms
   - Cost: ₪0 (бесплатный API)

2. **Full Report (Полный отчёт)**
   - Владельцы, директора (из Companies Registrar)
   - Статус Osek Murshe (из Shaam)
   - Задолженность по годовым сборам
   - Latency: 1-3 секунды
   - Cost: ₪11-36 (если нужна полная выписка)

**Технологический стек:**
- Python 3.11 + FastAPI
- Redis для кэширования (TTL: 24 часа)
- PostgreSQL для логирования запросов

---

#### Module 2: Financial Risk (Финансовый риск)

**Источники:**
- Bank of Israel (Ограниченные счета) — файлы
- BDI Code API — кредитный скоринг

**Функции:**
1. **Restricted Accounts Check**
   - ETL: Ежедневная загрузка ZIP → парсинг → Redis
   - Input: ID number (company/individual)
   - Output: Boolean + дата ограничения
   - Latency: <100ms (in-memory Redis)

2. **Credit Score (через BDI)**
   - Input: H.P. number
   - Output: BDI Rating, Red Lights, FICO score
   - Latency: 2-5 секунд (BDI API call)
   - Cost: $0.50-2 per query (Tier 1 contract)

**Технологический стек:**
- Python Celery для ETL (daily cron)
- Redis (in-memory) для мгновенного поиска
- BDI Code SDK для API интеграции

---

#### Module 3: Court History (Судебная история)

**Источники:**
- NetHaMishpat Gateway (Tier 1 партнёр)
- Takdin API (альтернатива)

**Функции:**
1. **Basic Lawsuit Check**
   - Input: Company name или owner name
   - Output: Количество исков, типы дел
   - Latency: 3-10 секунд
   - Cost: ₪5-10 per query (NetHaMishpat Gateway)

2. **User-Authorized Deep Dive**
   - User логинится через Hizdahut Leumit
   - SBF извлекает его дела (client-side session)
   - Latency: 10-30 секунд
   - Cost: ₪0 (пользователь авторизован)

**Технологический стек:**
- NetHaMishpat Gateway REST API
- OAuth2 proxy для user-authorized access
- PostgreSQL для хранения результатов (с TTL)

---

#### Module 4: User Finance (Open Banking)

**Источники:**
- WeCheck API (партнёр)
- Будущее: Прямые банковские API (с лицензией)

**Функции:**
1. **Cash Flow Analysis (Анализ денежных потоков)**
   - User подключает банковский счёт через WeCheck
   - SBF получает транзакции за 3-6 месяцев
   - ML-модель оценивает стабильность доходов
   - Output: Stability Score (0-100)

**Применимость:** Премиум feature для проверки ИП/фрилансеров

**Технологический стек:**
- WeCheck SDK
- Python Pandas для анализа транзакций
- scikit-learn для ML-скоринга

### 3.3. Кэширование и оптимизация

**Стратегия кэширования:**

| Данные | TTL | Причина |
|--------|-----|---------|
| Companies Registrar (базовые) | 24 часа | Статус меняется редко |
| Shaam Osek status | 6 часов | Может измениться в течение дня |
| BDI Credit Score | 7 дней | Обновляется еженедельно |
| Court data | 30 дней | Новые дела появляются медленно |
| BOI Restricted Accounts | 24 часа | Ежедневное обновление |

**Экономия на кэшировании:**
- Запрос к BDI: $0.50-2
- Повторный запрос (из кэша): $0
- При 30% повторных запросов: экономия ~$300/месяц на 1000 запросах
