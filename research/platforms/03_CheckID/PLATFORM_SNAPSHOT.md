## 1. Общая информация

### Базовые данные
- **Официальное название:** Guideline Group Technologies Ltd. (גיידליין גרופ טכנולוגיות מידע בע"מ)
- **Бренд:** CheckID  
- **Сайт:** https://www.checkid.co.il  
- **Модель:** Freemium + Pay-per-query API

### Корпоративная структура
- **Материнская компания:** Guideline Group Technologies — крупнейший израильский поставщик бизнес-данных
- **Портфолио Guideline:** CheckID, Takdin (суды), Tabu.co.il (недвижимость), Nevo (законодательство)

- **Покрытие:** 500,000+ израильских компаний + физлица + транспорт + недвижимость

**Ключевое отличие:** CheckID только **агрегирует** публичные данные, не создаёт кредитные рейтинги.

### 2.2. Детальная карта государственных источников

#### **Источник 1: רשם החברות (Companies Registrar)**

**Тип доступа:** ✅ Прямой API через data.gov.il

**Данные:**
- Company ID (H.P. number)
- Hebrew/English names
- Status (Active/Dissolved/Violating)
- Владельцы (בעלי מניות) с процентами
- Директора (דירקטורים)
- История изменений
- Адрес регистрации

**Метод интеграции:**
```
[CheckID] → data.gov.il CKAN API (REST/JSON)
          → Платные запросы к רשם החברות (₪7 за Nesach)
          → Real-time queries (нет кэша — всегда свежие данные)
```

**API Endpoint:** `/exApi/v1/CheckId/GetData/RashamHavarotDataModel`

**Цена агрегации CheckID:**
- Basic Company Info: ₪0 (бесплатный API data.gov.il)
- Full Nesach (выписка): ₪7 агрегация + ₪12 государственная пошлина = **₪19 total**

**Частота обновления:** Real-time (каждый запрос = новый запрос к государству)

#### **Источник 2: רשות המסים (Tax Authority — Shaam)**

**Тип доступа:** ✅ API к публичным данным (עוסק מורשה статус)

**Данные:**
- Статус עוסק מורשה/פטור (зарегистрирован ли как плательщик НДС)
- Номер עוסק (Business Tax ID)
- Дата регистрации
- Статус ניכוי במקור (удержание налога у источника)

**Метод интеграции:**
```
[CheckID] → Tax Authority Public API
          → OAuth2 authentication (CheckID зарегистрирован как "Service Provider")
          → Query by H.P. number or Tax ID
```

**API Endpoints:**
- `/exApi/v1/CheckId/GetData/MaamDataModel` — статус עוסק מורשה
- `/exApi/v1/CheckId/GetData/NikuiDataModel` — ניכוי במקור

**Цена агрегации CheckID:**
- Статус עוסק: **₪0.50** (CheckID markup за агрегацию)
- ניכוי במקור: **₪0.50**

**Частота обновления:** Real-time

**Ограничения:**
- ❌ НЕ получает налоговые декларации (Form 1214)
- ❌ НЕ получает данные о доходах/расходах
- ✅ Только публичный статус регистрации

#### **Источник 3: בנק ישראל (Bank of Israel) — Restricted Accounts**

**Тип доступа:** ✅ Публичный файл (Hashbonot Mugbalim)

**Данные:**
- Список компаний/физлиц с ограниченными счетами
- Причина ограничения (10+ возвращённых чеков)
- Дата начала ограничения
- Банк, наложивший ограничение

**Метод интеграции:**
```
[CheckID] → Daily download ZIP from boi.org.il
          → Parse ASCII/Excel (ISO-8859-8 encoding)
          → Local cache (обновление ежедневно)
```

**API Endpoint:** `/exApi/v1/CheckId/GetData/BoiDataModel`

**Цена агрегации CheckID:** **₪0.50** (за запрос к кэшу CheckID)

**Частота обновления:** Daily (CheckID загружает файл каждое утро)

**Критическое отличие от BDI Code:**
- ❌ CheckID **НЕ имеет доступа** к Credit Data Registry (требуется лицензия кредитного бюро)
- ❌ Нет данных о кредитной истории
- ❌ Нет Trade Payment Data
- ✅ Только публичный список Mugbalim

### 2.3. ЧТО CheckID НЕ ИМЕЕТ (vs кредитные бюро)

**Критические ограничения:**

❌ **Bank of Israel Credit Data Registry:**
- Нет доступа к кредитной истории (требуется лицензия кредитного бюро ₪500K-4M капитал)
- Нет данных о кредитах
- Нет Trade Payment Data (как компания платит поставщикам)
- Нет FICO®BDI / PAYDEX® / BdiCode Rating

❌ **Кредитный скоринг:**
- CheckID **НЕ создаёт** кредитные рейтинги
- Нет прогноза банкротства (PD Score)
- Нет оценки риска (Risk Score)

❌ **Финансовая аналитика:**
- Нет оценки оборота (Revenue Estimate)
- Нет прибыльности
- Нет сравнения с отраслью

**Вывод:** CheckID — это **"сырые данные"** от государства, без аналитики кредитного бюро.
